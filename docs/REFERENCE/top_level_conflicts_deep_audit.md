## Phase 3: Proposed Target Structure

### Canonical Structure Rules

1. **Documentation:** All docs live under `docs/`
   - `docs/FEATURES/` - Feature documentation
   - `docs/REFERENCE/` - Reference documentation
   - `docs/OPERATIONS/` - Operations guides
   - `docs/ARCHIVE/` - Archived/dated documentation (YYYY-MM-DD prefix for dated docs)
   - `docs/DEVELOPMENT/` - Development guidelines
   - `docs/assets/` - Documentation assets (images referenced by docs)

2. **Frontend Public Assets:** Live under `front-end/public/`
   - HTML pages, CSS, JS, images for frontend
   - Served by frontend build system
   - PWA manifests, service workers

3. **Root Public Assets:** Live under `public/` (root)
   - Standalone HTML pages served directly by Nginx
   - Shared assets not part of frontend build
   - Static files for direct access

4. **Generated Artifacts:** Live under `logs/` or `reports/`
   - `logs/` - Runtime logs, build logs (organized by source: `logs/build/`, `logs/server/`, `logs/front-end/`)
   - `reports/` - Generated reports, analysis outputs, test artifacts

5. **Tooling:** Live under `tools/` or `scripts/`
   - Development tools and utilities
