## Overview

The Architecture Audit reports 1059 issues for OrthodoxMetrics across six violation types. After triage, **roughly two-thirds of those items are false positives** generated by overly broad rules that fire on utility files, type declarations, theme files, and child components that don't fetch their own data. This plan first **fixes the audit rules** (one PR in OMAI), then resolves the **388 real items** in batches.

### Goals

- Eliminate the 671 false-positive items by tightening the scanner rules
- Resolve every legitimate `ROGUE_API_CLIENT` and `HARDCODED_COLORS` violation through small mechanical PRs
- Refactor the 40 real `STATE_EXPLOSION` components to use `useReducer`, custom hooks, or split state
- Wire real API integration into the 69 `NO_API_CLIENT` route pages that legitimately need it
- Investigate and triage the 31 ambiguous `PLACEHOLDER_STUB` items
- Resolve 3 architectural `CROSS_FEATURE_IMPORT` issues

### Non-Goals

- No new features
- No performance work or design changes beyond what's needed to pass the rule
- No mass-rewriting of files that overlap with the active God Component refactor — those auto-resolve when their parent god component is finished

## The Audit Rule Problem

The scanner at `/var/www/omai/_runtime/server/src/api-ops/architecture-audit.js` applies these rules to every `.tsx`/`.ts`/`.jsx`/`.js` file under `front-end/src/`:

```js
if (m.apiCount === 0 && m.loc > 50)        v.push({ code: 'NO_API_CLIENT', ... });
if (m.apiClient === 'fetch')               v.push({ code: 'ROGUE_API_CLIENT', ... });
if (m.isPlaceholder)                       v.push({ code: 'PLACEHOLDER_STUB', ... });
if (m.hardcodedColors > 10)                v.push({ code: 'HARDCODED_COLORS', ... });
if (m.useStateCount > 15)                  v.push({ code: 'STATE_EXPLOSION', ... });
```

The first four rules don't distinguish between **route-mounted page components** (which legitimately should fetch data, use theme tokens, and not be stubs) and **utilities, types, theme files, hooks, and child components** (which legitimately should not). This produces:

| Violation | Total | False positives | Reason |
|-----------|-------|----------------|--------|
| `NO_API_CLIENT` | 699 | **630** | Fires on utils/types/theme/hooks/config/layouts/shared/ui and child components that receive data via props |
| `PLACEHOLDER_STUB` | 59 | 28 | Fires on tiny re-export `index.ts` files, type declaration files, hook stubs |
| `HARDCODED_COLORS` | 107 | 9 | Fires on theme files that **define** the color tokens, type files containing color enums |
| `ROGUE_API_CLIENT` | 143 | 4 | Fires on `utils/logger` and `utils/authErrorHandler` which intentionally use `fetch` to avoid circular deps |
| `STATE_EXPLOSION` | 48 | 0 | Real signal — no false positives |
| `CROSS_FEATURE_IMPORT` | 3 | 0 | Real signal — no false positives |
| **Total** | **1059** | **671** | |

Phase 0 fixes the rules. The remaining 388 items are real and broken down per phase below.

### Changes

Add a helper near the top of the file:

```js
// Directories that contain non-UI code (utils, types, theme, etc.)
// Files in these directories should be exempt from UI-component rules
const NON_UI_GROUPS = new Set([
  'utils', 'types', 'theme', 'config', 'hooks', 'routes',
  'context', 'lib', 'api', 'auth', 'mocks', 'sandbox', 'ai', 'store', 'ui'
]);

// Files that intentionally use raw fetch to avoid circular dependencies
const ROGUE_FETCH_ALLOWLIST = new Set([
  'utils/logger', 'utils/authErrorHandler'
]);

function isNonUiFile(relPath) {
  const firstSeg = relPath.split('/')[0];
  if (NON_UI_GROUPS.has(firstSeg)) return true;
  if (relPath.startsWith('@om/components/ui/')) return true;
  if (relPath.startsWith('shared/ui/') || relPath.startsWith('shared/lib/')) return true;
  if (firstSeg === 'layouts') return true;
  return false;
}

function isRoutePage(relPath) {
  const fname = relPath.split('/').pop().replace(/\.[^.]+$/, '');
  return /(?:Page|Dashboard|Console|View)$/.test(fname) || relPath.includes('/pages/');
}
```

Then update `detectViolations(name, m)` to accept the `relPath` and gate the four broken rules:

```js
function detectViolations(name, m, relPath) {
  const v = [];
  const nonUi = isNonUiFile(relPath);

  if (m.isGod) v.push({ code: 'GOD_COMPONENT', sev: 'critical', title: `${name}: God Component (${m.loc} LOC)` });
  if (m.isMock) v.push({ code: 'MOCK_DATA', sev: 'critical', title: `${name}: Mock/hardcoded data` });
  if (m.emptyCatch > 0) v.push({ code: 'EMPTY_CATCH', sev: 'high', title: `${name}: ${m.emptyCatch} empty catch blocks` });

  // NO_API_CLIENT: only flag route-mounted pages, never util/type/theme/hook files
  if (m.apiCount === 0 && m.loc > 50 && !nonUi && isRoutePage(relPath)) {
    v.push({ code: 'NO_API_CLIENT', sev: 'high', title: `${name}: No API integration` });
  }

  // ROGUE_API_CLIENT: skip files on the allowlist + skip api/auth groups
  if (m.apiClient === 'fetch' && !ROGUE_FETCH_ALLOWLIST.has(relPath.replace(/\.[^.]+$/, ''))
      && !relPath.startsWith('api/') && !relPath.startsWith('auth/')) {
    v.push({ code: 'ROGUE_API_CLIENT', sev: 'high', title: `${name}: Uses raw fetch()` });
  }

  // PLACEHOLDER_STUB: skip non-UI files and re-export index files
  if (m.isPlaceholder && !nonUi && !relPath.endsWith('/index.ts') && !relPath.endsWith('/index.tsx')) {
    v.push({ code: 'PLACEHOLDER_STUB', sev: 'high', title: `${name}: Placeholder stub` });
  }

  if (m.crossFeature > 5) v.push({ code: 'CROSS_FEATURE_IMPORT', sev: 'medium', title: `${name}: ${m.crossFeature} cross-feature imports` });
  if (m.useStateCount > 15) v.push({ code: 'STATE_EXPLOSION', sev: 'high', title: `${name}: ${m.useStateCount} useState calls` });

  // HARDCODED_COLORS: skip theme/types/store/context files
  if (m.hardcodedColors > 10 && !['theme','types','store','context'].includes(relPath.split('/')[0])
      && !relPath.startsWith('@om/components/ui/theme/')) {
    v.push({ code: 'HARDCODED_COLORS', sev: 'medium', title: `${name}: ${m.hardcodedColors} hardcoded colors` });
  }

  if (m.pollingMs && m.emptyCatch > 0) v.push({ code: 'NO_CLEANUP', sev: 'medium', title: `${name}: Polling without cleanup` });
  return v;
}
```

Update the call site (one line) to pass `relPath`:
```js
const violations = detectViolations(name, metrics, file.relPath);
```

## Phase 1 — ROGUE_API_CLIENT

**Owner**: Windsurf
**Real items**: 127 (after Phase 0 + excluding 12 already inside the God Component refactor)
**Pattern**: Replace raw `fetch(url, opts)` with `apiClient.get/post/put/delete(url, opts)`

### Why it's safe to batch

`apiClient` (in `front-end/src/api/utils/axiosInstance.ts`) is a thin wrapper around axios that auto-prefixes `/api`, attaches the bearer token, and handles 401 redirects. Replacing `fetch` with `apiClient` is a 1-to-1 mechanical transformation in almost every case. Edge cases (multipart uploads, streaming responses) need special handling — flag them and skip.

### Refactoring pattern

```ts
// BEFORE
const res = await fetch('/api/foo/bar', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` },
  body: JSON.stringify(payload),
});
const data = await res.json();

// AFTER
import apiClient from '@/api/utils/axiosInstance';
const { data } = await apiClient.post('/foo/bar', payload);
```

Notes:
- Drop the leading `/api` (axiosInstance adds it)
- Drop the `Authorization` header (auto-attached)
- Drop the `Content-Type` header for JSON (axios sets it)
- For multipart/FormData, use `apiClient.post(url, formData, { headers: { 'Content-Type': 'multipart/form-data' } })`

### Pattern: useReducer extraction

```ts
// BEFORE: 18 individual useState
const [name, setName] = useState('');
const [email, setEmail] = useState('');
const [age, setAge] = useState(0);
// ... 15 more

// AFTER: single useReducer
type FormState = { name: string; email: string; age: number; /* ... */ };
type FormAction =
  | { type: 'SET_FIELD'; field: keyof FormState; value: any }
  | { type: 'RESET' };

const [form, dispatch] = useReducer((state: FormState, action: FormAction) => {
  switch (action.type) {
    case 'SET_FIELD': return { ...state, [action.field]: action.value };
    case 'RESET': return initialFormState;
  }
}, initialFormState);

// Usage
<TextField value={form.name} onChange={e => dispatch({ type: 'SET_FIELD', field: 'name', value: e.target.value })} />
```

For data-fetching state, prefer extracting into a custom hook:

```ts
// hooks/useChurchOnboardingPipeline.ts
export function useChurchOnboardingPipeline(id: string) {
  const [data, setData] = useState<Pipeline | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => { /* fetch */ }, [id]);
  return { data, loading, error, refetch };
}
```
