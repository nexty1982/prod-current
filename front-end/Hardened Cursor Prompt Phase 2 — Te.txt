Part A — Data Model: store which templates each church selected

Create a table in orthodoxmetrics_db if missing:

church_record_templates

id (PK)

church_id (INT, indexed)

record_type ENUM('baptism','marriage','funeral','custom')

template_slug VARCHAR(100)

language VARCHAR(10) NULL (optional)

is_active TINYINT(1) default 1

created_at / updated_at

UNIQUE(church_id, record_type)

This table is the permanent source of truth for what a church uses.

Part B — Templates.fields JSON contract (minimum viable)

Templates are stored in orthodoxmetrics_db.templates.fields (JSON in LONGTEXT). Provisioning must validate:

Required:

version: number

columns: Array<{ name: string, sqlType: string, nullable?: boolean, default?: any, primaryKey?: boolean, autoIncrement?: boolean }>

Optional:

indexes: Array<{ name: string, columns: string[], unique?: boolean }>

engine?: string

charset?: string

Rules:

name must be safe identifier (snake_case, letters/numbers/_)

sqlType must be from an allowlist (VARCHAR/INT/DATE/DATETIME/TEXT/LONGTEXT/JSON/TINYINT/DECIMAL/etc.)

Reject templates missing required fields.

Part C — Backend: provisioning implementation
1) Add TemplateProvisioning service

Create:

server/services/templateProvisioningService.js

Responsibilities:

Load template by slug from orthodoxmetrics_db.templates

Parse & validate fields JSON

Generate CREATE TABLE IF NOT EXISTS SQL for the church DB table:

baptism → baptism_records

marriage → marriage_records

funeral → funeral_records

Execute SQL via getChurchDbConnection(database_name) (do not use getAppPool for church DB writes) 

BACKEND_FIELD_MAPPER_PROVISIONI…

Apply indexes

Return structured result: created tables, warnings, errors

2) ensureRecordTablesFromTemplates(churchId, database_name, selection)

Inputs:

churchId

database_name

selection: { baptism?: slug, marriage?: slug, funeral?: slug }

Behavior:

For each record_type in selection:

Load template by slug

Create the corresponding *_records table

If table exists: do not drop, do not alter (Phase 2 is create-only)

If invalid template or SQL failure: mark failed

Fallback:

For any failed record_type(s), call existing hardcoded schema creation ONLY for those missing tables (do not recreate successful tables).
The existing hardcoded schema function exists today (createChurchDatabaseSchema) and must remain as fallback.
