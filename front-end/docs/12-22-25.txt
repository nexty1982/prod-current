STEP 2 — CREATE A TRANSCRIPTION NORMALIZATION MODULE (SERVER)
========================================================
Create a new module (server-side) that converts the existing OCR response into a readable transcription.

Create: server/src/ocr/transcription/normalizeTranscription.ts

Types:
- Define `OcrToken` and `OcrLine` types explicitly.
- Prefer normalized image coords for bbox: { x0,y0,x1,y1 } in [0..1].
- Token: { text: string; confidence?: number; bbox?: BBox; langHint?: 'latin'|'cyrillic'|'mixed'|'unknown' }

Function signature:
- `normalizeTranscription(input: { tokens: OcrToken[]; lines?: OcrLine[]; }): { text: string; paragraphs: string[]; diagnostics: {...} }`

RULES (MUST IMPLEMENT)
A) Script detection (critical)
- Implement a fast script classifier:
  - If a string contains any Cyrillic unicode range: script='cyrillic'
  - Else if contains A-Z/a-z: script='latin'
  - Else 'unknown'
- Never “correct” Cyrillic into Latin-like junk.
- Do not merge tokens of different scripts into the same word unless they are clearly numeric/punctuation.

B) Token cleanup
- Trim, collapse whitespace.
- Drop tokens that are:
  - length=1 and not a letter/number (punct-only)
  - OR confidence < threshold (default 0.35) AND length <= 2
  - OR match common OCR garbage regex (e.g., repeated random consonants) — keep this conservative.

C) Line reconstruction
- If the OCR provider already gives lines: use them.
- If not, reconstruct lines by clustering tokens by Y:
  - Sort tokens by y-center
  - Group into “line bands” where y-centers differ less than an adaptive threshold (based on median token height).
  - Within each line, sort by x0 and join with spaces.
- Preserve hyphenated line breaks:
  - If line ends with '-' and next line starts with a letter: join without space and remove '-'.

D) Paragraph reconstruction
- Split paragraphs by vertical gaps:
  - Compute y-gap between adjacent lines.
  - If gap > (median line height * 1.2) start a new paragraph.
- Also start a new paragraph when script changes (latin<->cyrillic) and the line begins with a capitalized label-like token or a short header.

E) Readability formatting
- Output text should be paragraphs separated by a blank line.
- Within paragraphs keep line breaks only if they look intentional (e.g., header blocks); otherwise wrap lines into a single paragraph line.

Diagnostics
Return:
- droppedTokenCount
- lineCount
- paragraphCount
- scriptsPresent
- warnings (e.g., “low_confidence_overall”)
